# Скрипты для консолидации данных об участии сотрудников в работе над проектами
## Назначение файлов
 - ``projects.sqlite`` файл с БД SQLite
 - ``consolidation_database.sql`` скрипт разворачивания БД
 - ``projects.csv`` тестовый набор данных
 - ``run_consolidation.py`` скрипт консолидации
 - ``output/report.csv`` консолидированные данные 
# Инструкция по эксплуатации
## Важные замечания 
Скрипты не обрабатывают ошибки, не проверяют входные данные, при сбоях в работе следует обратиться к поставщику 
## Установка                                      
 - из репозитория скопировать директорию ``tensor-employee\csv-consolidation`` (файлы projects.csv и output/report.csv можно проигнорировать)
 - установить Python 3.7.3 и SQLite 3.25.1
 - выполнить скрипт ``consolidation_database.sql`` 
## Текущее обслуживание 
Перед каждым запуском скрипта следует очищать таблицы contributor и project
```
DELETE FROM contributor WHERE 1=1
DELETE FROM project WHERE 1=1
```
### PS
Можно было для каждого запуска скрипта записывать в БД номер запуска консолидации, что бы отличать свежие данные от старых, но хороша ложка к обеду 
# Руководство пользователя
 - разместить ``*.csv`` файлы в директрии со скриптом ``run_consolidation.py`` (``\csv-consolidation``)
 - выполнить скрипт ``run_consolidation.py``
 - консолидированные данные будут размещены в файле ``output/report.csv``
 
Файл ``output/report.csv`` при каждом запуске скрипта перезаписывается, перед запуском скрипта при необходимости или переименуйте файл с результатом консолидации или скопируйте его в другую директорию
